# ☕ Bright Coffee Shop Sales Analysis  
*A Data-Driven Business Intelligence Project*

---

## 📌 Project Overview

This project analyzes transactional data from a coffee shop to uncover insights that improve **sales performance, customer experience, and operational efficiency**.

Using **SQL and Excel (Pivot Tables & Dashboards)**, the dataset is explored to identify patterns in revenue, product performance, customer behavior, and time-based trends.

The goal is to transform raw data into **actionable business recommendations**.

---

## 🎯 Objectives

- Understand overall **sales performance**
- Identify **top-performing products and categories**
- Analyze **customer purchasing behavior**
- Determine **peak sales periods (time & day)**
- Compare performance across **store locations**
- Provide **data-driven recommendations**

---

## 📂 Dataset Description

The dataset contains transactional records with the following fields:

- **Transaction ID** – Unique identifier for each transaction  
- **Transaction Date** – Date of purchase  
- **Transaction Time** – Time of purchase  
- **Store Location** – Branch where the purchase occurred  
- **Product Category** – e.g., Coffee, Tea, Snacks  
- **Product Detail** – Specific product name  
- **Transaction Quantity** – Number of items sold  
- **Revenue** – Total amount per transaction

## 🔍 Data Analysis Process

### 1. Data Exploration
- Reviewed dataset structure using SQL  
- Checked for missing values and inconsistencies  
- Verified date ranges  

### 2. Data Transformation
Created calculated fields:
- Month Name  
- Day of Week  
- Time Classification (Morning, Afternoon, Evening)  
- Spend Categories  

### 3. Data Aggregation
- Summarized revenue, quantity, and transactions using pivot tables  

### 4. Data Visualization
- Built charts and an interactive dashboard using slicers
